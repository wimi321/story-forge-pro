# Security Policy

## Reporting

If you discover a security issue, do not post it in a public issue.
Please report it privately through GitHub security reporting if enabled, or contact the repository owner directly.

## Scope

Security issues include:

- credential leaks
- insecure launch behavior
- unsafe command execution guidance
- packaging or dependency risks

## User Responsibility

Users should never commit `.env`, `.env.local`, API keys, or personal manuscript data they do not want shared.
